## 🎯 工作原理

```
访问详情页
   ↓
auth.js 检查 localStorage 中的 Token
   ↓
Token 存在且有效？
   ├─ 是 → 正常加载页面 ✓
   └─ 否 → 重定向到 login.html 🔐
   
登录表单
   ↓
输入凭证，点击登录
   ↓
验证成功？
   ├─ 是 → 生成 Token → 存储 → 返回原页面 ✓
   └─ 否 → 显示错误提示
```

### 演示版本说明
当前版本是**演示/开发版本**，生产环境需要：

1. **后端验证** - 将认证逻辑移到后端服务器

## 📞 故障排除

### Q: 登录后仍然跳转到登录页？
**A:** 
- 检查浏览器是否禁用了 localStorage
- 开发者工具 → Application → Storage → Local Storage
- 确保有 `authToken` 和 `loginTime` 两个字段

### Q: 登出后无法再登录？
**A:** 
- 这不应该发生，可能是缓存问题
- 按 Ctrl+Shift+Delete 清除浏览器缓存后重试

### Q: 能否保持登录状态超过 24 小时？
**A:** 
- 可以，编辑 `auth.js` 中的 `TOKEN_EXPIRY_MS`
- 改为：`48 * 60 * 60 * 1000` 为 48 小时等
