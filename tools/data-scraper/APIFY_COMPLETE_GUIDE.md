# Apify 完整爬虫指南 - BYD AUTO BRASIL

## 📋 任务目标

- ✅ 爬取全部 **189 条视频**（而不是之前的 100 条）
- ✅ 爬取每个视频的前 **10 条高赞评论**
- ✅ 按赞数排序评论

## 🚀 两步执行流程

### 第一步：爬取全部视频（~5-15 分钟）

```bash
node --env-file=.env apify_videos_only.js
```

**功能说明:**
- 使用 `clockworks/tiktok-scraper`（更强大的通用爬虫）
- 目标爬取 300 个结果，但最多会获取 189 条
- 保存输出: `videos_all.json` (189 个视频)
- 成本: 约 500-1000 积分

### 第二步：爬取所有视频的高赞评论（20-60 分钟）

```bash
node --env-file=.env apify_comments_scraper.js
```

**功能说明:**
- 读取第一步的 `videos_all.json`
- 逐批爬取评论（每批 10 个视频）
- 每个视频获取前 10 条高赞评论
- 按赞数自动排序
- 保存输出: `top_comments.json`
- 成本: 约 1900 积分（189 个视频 × 10 条评论）

## ⚠️ 重要提示

### 积分消耗估计
- **视频爬取**: 500-1000 积分
- **评论爬取**: ~1900 积分（189 × 10）
- **总计**: 约 2400-2900 积分

### 时间估计
- **视频爬取**: 5-15 分钟
- **评论爬取**: 20-60 分钟（包括速率限制延迟）
- **总计**: 约 30-75 分钟

### 注意事项
1. 评论爬取会消耗大量积分，请确保账户有充足额度
2. 脚本中有 2 秒延迟以避免触发反爬虫
3. 如果某个视频无法访问，脚本会自动跳过
4. 评论按赞数排序，高赞评论优先显示

## 🔧 可选配置修改

### 减少积分消耗
```javascript
// 在 apify_comments_scraper.js 中修改
maxComments: 5,  // 改为 5 条而不是 10 条
```

### 加快速度（风险更高）
```javascript
// 减少延迟（秒）
await new Promise(r => setTimeout(r, 1000));  // 从 2000 改为 1000
```

## 🆘 常见问题

**Q: 为什么不能一次性爬取视频和评论？**
A: 为了避免请求过多导致被限流，分两步执行更稳定。

**Q: 如果爬取中途失败怎么办？**
A: 脚本会跳过失败的视频继续执行，之后可以重新运行补充。

**Q: 怎样获取所有评论而不仅是前10条？**
A: 修改脚本中的 `maxComments: 10` 为更大的数值（但会消耗更多积分）。

**Q: 积分不足怎么办？**
A: 减少 `maxComments` 或在 https://console.apify.com 充值。
